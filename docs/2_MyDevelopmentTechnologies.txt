-	Version control system

		1. Git

			a.) Deploy a successfull git branching model:

				a.1) Master branch
				a.2) Develop branch (when mature enought, merge into master,
				     tag with release number)
				a.3) Feature branches (limited life time)
					a.3.1) may branch off from DEVELOP
					a.3.2) must merge back into DEVELOP
					a.3.3) branch naming convention: anything
				a.4) Release branches (limited life time)
					a.4.1) may branch off from DEVELOP
					a.4.2) must merge back to DEVELOP and MASTER
					a.4.3) branch naming convention: release-*
				a.5) Hotfix branches (limited life time)
